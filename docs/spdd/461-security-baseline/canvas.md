## R — Requirements

**Problem:** The 2026-05 architectural review rates security at 3.5/10. Three gaps drive this EPIC:
1. The api-gateway accepts `POST /api/v1/apply` and `DELETE /api/v1/workflows/{id}` with no authentication — a reachable instance is a remote workflow-execution facility open to anyone. (Review C6, §8.2 S1)
2. Event publish errors in engine-adapter are silently discarded with `_ = pub.Publish(...)`. Operators discover bus failures only via missing dashboard data. (Review §5.7)
3. No request correlation ID flows from api-gateway through downstream services. Incident forensics require matching timestamps rather than a shared ID. (Review M4, §8.2 S7)

**Definition of done:**
- `POST /api/v1/apply` without `Authorization: Bearer <key>` returns `401 Unauthorized`.
- Event publish failures appear in structured logs as `level=WARN event=<name> err=<err>`.
- Log lines for a single `zynax apply` across api-gateway, workflow-compiler, and engine-adapter share the same `request_id` value.

---

## E — Entities

- **Bearer-token middleware** — HTTP middleware in api-gateway that validates `Authorization: Bearer <key>` against `ZYNAX_API_KEY` env var.
- **`ZYNAX_API_KEY`** — env var controlling auth gate; empty = disabled (dev mode, logs a WARN).
- **`pub.Publish()`** — event bus publish call in engine-adapter (`interpreter.go` and `temporal_workflow.go`); currently `_ = pub.Publish(...)`.
- **`slog.Warn`** — Go structured logger call replacing the discarded error.
- **`X-Request-ID`** — HTTP request header; generated by api-gateway if absent; propagated as gRPC metadata key `request-id`.
- **gRPC `UnaryServerInterceptor`** — extracts `request-id` from metadata and stores in context for downstream use.
- **`grpc.UnaryClientInterceptor`** — injects `request-id` from context into outgoing gRPC metadata.

---

## A — Approach

**What we WILL do:**
- Add HTTP middleware in api-gateway: check `Authorization: Bearer` header on mutating endpoints. Return 401 if absent/wrong when `ZYNAX_API_KEY` is set.
- Replace all `_ = pub.Publish(...)` calls with error-checking + `slog.Warn`.
- Add X-Request-ID generation and propagation through gRPC metadata.

**What we WON'T do:**
- Implement JWT, OIDC, or RBAC (M6+ concerns).
- Add rate limiting (M6+).
- Wire Prometheus counters for event failures here (deferred to M7.A).
- Add mTLS between services (M6.B).

**ADR references:**
- ADR-016: Layered testing — unit tests required for middleware.
- ADR-009: Language strategy — all changes are in Go services.

### Feature Safeguards
- Never log the value of `ZYNAX_API_KEY` — only its presence/absence.
- Never disable the auth gate in production code paths — only via the empty-key dev-mode convention.
- Never hardcode a default API key value anywhere in source code.
- The middleware must not affect GET endpoints (read-only paths remain open until M6 OIDC).
