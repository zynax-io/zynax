## R — Requirements

**Problem:** All inter-service gRPC connections use `insecure.NewCredentials()` (review §8.2 S3). For a control plane that executes workflows on behalf of tenants, unauthenticated inter-service communication is a defense-in-depth gap. A compromised agent could impersonate a platform service on the same network segment.

**Definition of done:**
- All gRPC connections between platform services use mutual TLS.
- A connection with an expired or wrong certificate is rejected at the TLS handshake.
- `make run-local` starts successfully with auto-generated dev certificates.
- `ZYNAX_TLS_*` env vars documented in infra/AGENTS.md.

---

## E — Entities

- **`ZYNAX_TLS_CERT`** — env var: path to the service's TLS certificate (PEM).
- **`ZYNAX_TLS_KEY`** — env var: path to the service's private key (PEM).
- **`ZYNAX_TLS_CA`** — env var: path to the CA certificate bundle used to verify peer certs (PEM).
- **`credentials.NewTLS(tlsConfig)`** — Go gRPC TLS credentials; replaces `insecure.NewCredentials()`.
- **Dev CA** — self-signed CA generated by an init container or script in the dev compose environment; certs written to a shared volume.
- **cert-manager `Certificate` + `Issuer`** — K8s resources for Helm-based production deployments.

---

## A — Approach

**What we WILL do:**
- Add `ZYNAX_TLS_CERT`, `ZYNAX_TLS_KEY`, `ZYNAX_TLS_CA` env vars to all services; build `tls.Config` from them.
- Replace `insecure.NewCredentials()` with `credentials.NewTLS(tlsConfig)` in all gRPC clients.
- Replace plain `grpc.NewServer()` with `grpc.NewServer(grpc.Creds(...))` in all servers.
- Dev compose: add an `openssl`-based cert-gen init container writing certs to a shared volume.
- Helm chart: add cert-manager `Certificate` and `Issuer` resources.

**What we WON'T do:**
- Add external (client-facing) TLS to the api-gateway HTTP port — that belongs at the ingress/LB.
- Implement SPIFFE/SPIRE (beyond scope for M6; evaluate in M7+).

**ADR references:**
- ADR-001: gRPC as inter-service protocol — TLS is an extension of the transport, not a contract change.
- ADR-009: Language strategy — cert-gen init container can use shell/openssl.
